Ejemplo práctico: mini página renderizada

Esta es la última fase: el navegador ya interpretó el HTML y muestra la página. Abajo hay una mini página de ejemplo que reúne los elementos básicos de HTML que vimos: imagen, lista, texto, tabla y formulario.

Una imagen ilustrativa

El navegador convierte cada etiqueta del código en un elemento visual. Este esquema muestra cómo el código se transforma en la página que vemos:

Texto y lista de contenidos

Las páginas combinan texto con elementos de estructura. Por ejemplo, una lista organiza la información de forma clara. Estos son los elementos básicos que suele tener una página:

  • Texto: títulos (<h1> a <h6>) y párrafos (<p>).
  • Imágenes: con la etiqueta <img>.
  • Enlaces: hipervínculos con <a href>.
  • Listas: ordenadas <ol> y no ordenadas <ul>.
  • Tablas y formularios: para datos y entrada de información.

Una tabla de datos

Las tablas muestran información en filas y columnas. Aquí un resumen de las tecnologías que intervienen en cada fase del proceso web:

Fase Tecnología Rol
Cliente HTML, CSS, JavaScript Estructura, estilo e interacción.
Comunicación HTTP Transporte de peticiones y respuestas.
Servidor Apache / XAMPP Procesa la solicitud y entrega recursos.
Lógica PHP, Python, Node.js Genera contenido dinámico.

Un formulario de ejemplo

Los formularios permiten enviar datos al servidor. Este formulario de contacto usa varios tipos de campos. Al enviarlo con el método POST, los datos viajarían en el cuerpo de la petición.





Proceso completo. Recorrimos las cinco fases: desde escribir la URL hasta ver una página renderizada con todos sus elementos. Este formulario, al enviarse, empezaría el ciclo otra vez con una nueva petición HTTP.
