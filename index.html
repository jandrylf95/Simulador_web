¿Qué ocurre al ingresar una URL?

Cada vez que escribimos una dirección en el navegador y presionamos Enter, se inicia un proceso de comunicación entre el navegador y un servidor. En este sitio vamos a recorrer ese proceso paso a paso.

El proceso en pocas palabras

La Web funciona con un modelo cliente - servidor. El navegador (cliente) pide un recurso y el servidor se lo entrega. Todo esto ocurre gracias al protocolo HTTP, que define la forma en que se piden y se entregan los documentos.

Las 5 fases que vamos a ver

Cada página de este sitio explica una fase del proceso. Conviene seguir el orden:

  1. El usuario ingresa una URL en el navegador.
  2. El navegador realiza una petición HTTP hacia el servidor.
  3. El servidor procesa la solicitud y prepara el recurso.
  4. Se envía una respuesta HTTP con el documento HTML.
  5. El navegador interpreta el HTML y muestra la página.

¿Qué es una URL?

Una URL (Uniform Resource Locator) es la dirección de un recurso en la Web. Está formada por varias partes:

  • Protocolo: https:// (indica cómo se comunican navegador y servidor).
  • Dominio: www.utpl.edu.ec (identifica al servidor).
  • Ruta: /carreras/index.html (señala el recurso específico).

Nota: la "W" representa la World Wide Web, que NO es un protocolo de correo, sino un sistema de documentos enlazados que se acceden mediante HTTP.
